      "htbw":"ht40",      // 频宽（ht20|ht40|auto）
      "htbw_autoreal":"ht40+" //实际具体频宽值 (ht20|ht40)
      "channel":"0"      // 0为自动(1~13)

      "htbw":"ht40",      // 频宽（ht20|ht40|ht80|auto）
      "htbw_autoreal":"ht40" // 实际具体频宽值(ht20|ht40|ht80)
      "channel":"36"      // 0, 36,40,44,48,149,153,157,161
      "channel_autoreal":"36" // 36,40,44,48,149,153,157,161 自动信道选择时，channel值为0，此时该值表示具体工作信道
    }
  }
 }

set_htbw (args)
设置频宽 High Throughput

Parameters:

  • args {device,htbw}

Usage:

     HT20是出于兼容性考虑：比如，一个区域内存在11b/g信号，那么为了尽量减少对它们的干扰，需要设定为HT20，以减少频带的重叠。
 HT40是出于高性能考虑：HT40相当于两个HT20的捆绑，一个是主，一个是辅。主信道发送beacon报文和部分数据报文，辅信道发送其他报文。

 args = {
  device="radio0"|"radio1"
  htbw="ht20"|"ht40"|"auto",               // 5G支持"ht80"
 }
 radio0 无线2.4G
 radio1 无线5G
